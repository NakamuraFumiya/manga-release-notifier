## 学んだこと
- `aws configure sso` の対話フロー: SSO session name → start URL → region → scopes → ブラウザ認証 → アカウント/ロール選択 → デフォルトリージョン/出力形式
- SSO start URL は IAM Identity Center のダッシュボードの「AWS access portal URL」で確認できる
- `AWS_PROFILE` 環境変数が設定されていると、`aws configure sso` がプロファイル名を聞かずにその既存プロファイルに書き込む → `--profile` で明示的に指定する必要がある
- プロファイル名はプロジェクト固有にせず汎用的な名前（`fumiya-personal`）にしておくと他の個人プロジェクトでも使い回せる
