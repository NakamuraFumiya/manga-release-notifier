## 学んだこと
- LocalStack の healthcheck エンドポイント (`/_localstack/health`) は LocalStack 独自の API で、curl で直接叩く
- 通常の AWS サービス操作は AWS CLI に `--endpoint-url=http://localhost:4566` を付けて叩く（例: `aws --endpoint-url=http://localhost:4566 sqs list-queues`）
- Go の AWS SDK (`aws-sdk-go-v2`) でも同様にエンドポイントを LocalStack に向けることで、本番と同じコードでローカル検証できる
- curl のレスポンスを整形するには `curl -s <url> | jq` でパイプする
