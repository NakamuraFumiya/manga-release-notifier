# 学習ログ作成スキル

## やること

1. 今日の日付で `docs/logs/YYYY-MM-DD.md` を作成する（既にあれば追記する）
2. 今回の会話で学んだことを整理して記録する

## ファイルのフォーマット

```markdown
# 学習ログ YYYY-MM-DD

## 今日やったこと
- （Phase 番号と作業内容）

## 学んだこと
- （新しく理解した概念や仕組み）

## 詰まったところ・疑問
- （ハマったポイントや未解決の疑問）

## 深掘りしたいこと
- （やり取りの中で理解が浅いと感じたポイント。次の作業とは別に、自分で調べたり試したりしておくとよいこと。各項目に参考リソースを添える：AWS公式ドキュメントのURL、手持ちの技術書の該当章など）

## 次にやること
- （次回の作業予定）
```

## ルール

- 既存ファイルがある場合は上書きせず、セクションに追記する
- 会話の内容から学びを抽出して書く（会話をそのままコピペしない）
- 簡潔に。箇条書き中心
- $ARGUMENTS が指定されていればその内容を重点的に記録する
